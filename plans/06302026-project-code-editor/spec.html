Why

DevEasy operators jump to an external editor to read or tweak a file in a project they're running a Claude session against. An in-app, VS Code–style editor keeps browse → edit → save in one window, right next to the session and git cockpit for that same repo. It also lets the operator view the images, PDFs, and rendered HTML that Claude produces without leaving the app.

What

A files icon next to the + in each project's Sessions-sidebar header. Clicking it takes over the Sessions panel for that project: the left rail becomes a lazy-loaded file tree (header shows the project name + a back arrow to return to sessions), and the main area becomes a VS Code–style tabbed editor.

  • Tabs: clicking a file opens a closeable tab to the right. Text files open in a Monaco editor (syntax highlighting + Emmet) and are fully editable.
  • Save & dirty state: Cmd/Ctrl+S writes to disk; a dirty tab shows a dot; closing a dirty tab prompts for confirmation.
  • Languages: JS/TS/JSX/TSX, JSON, .config files, XML, HTML, CSS, Markdown, and the rest of Monaco's built-in set, with Emmet on for HTML/JSX/CSS.
  • Binary viewers: images (PNG/JPG/GIF/SVG/WebP) and PDF render in the same tab system (read-only). HTML files toggle between view as code and view as page.
  • Persistence: open tabs + active tab persist per project across app reloads (DB-backed). Unsaved buffer content is not persisted.

Done when: from a project's files icon, the operator can browse the tree, open multiple file tabs, edit and save a text file to disk, view an image and a PDF, toggle an HTML file between code and page, close tabs (with an unsaved-changes guard), reload the app and find the same tabs reopened, and return to the session list via the back arrow.

  • Match the existing git-status query pattern in hooks/queries/useGit.ts for query-key shape and enabled gating.

Constraints

Must

  • Every file/dir path is validated server-side before any fs call: reject absolute paths and .., resolve against the project root, and verify the fs.realpath result is still inside the root (defeats symlink escape) (§5.2, §5.4).
  • Resolve the project root only via ProjectModel.findById — never trust a client-supplied root (§5.4, §7.4).
  • New backend domain mirrors git/ structure and the {success,data,error} contract (§6.1, §8.1).
  • Tree listing is lazy (one directory level per request) and skips .git and node_modules via a named ignore list (§4.2).
  • Text read/write capped by a named max size; binary stream capped by its own named max (§4.2).
  • Schema change ships as a Knex migration in the project's migrations dir (§10.3); the plan's migrations/ folder scaffolds it.
  • Monaco workers are self-hosted/bundled (offline), not loaded from a CDN — DevEasy is local-first (§4.4).
  • New deps justified and pinned: monaco-editor, @monaco-editor/react, emmet-monaco-es (§4.4).

Must Not

  • No inline db()/SQL outside models/ (§7.4); no business logic in routes/controllers (§7.2, §7.3).
  • No raw fetch in components/domain files — only api/index.ts (§14.2).
  • No dangerouslySetInnerHTML for HTML preview (§17.4); no any (§17.2); no console.* (§17.1).
  • Do not persist unsaved buffer contents to the DB (only which files are open).
  • Do not let the editor write outside the project root, follow a symlink out, or write to .git/.
  • No refactor of the existing Session chat flow beyond the takeover wiring.

Out Of Scope

  • Creating/renaming/deleting/moving files in the tree (read + edit-existing + save only for v1).
  • Multi-pane/split editor, diff view, search-across-files, git gutter decorations, terminal.
  • Non-JS-adjacent language servers / IntelliSense beyond Monaco's built-in tokenizer.
  • Editing binary files; office formats; very large files beyond the size cap.

Risk

Level: 3 — arbitrary file read/write under a project root is a real security surface, and Monaco is a heavy new dependency.

Risks Identified

  • Path traversal / symlink escape (read & write). A crafted path could read or overwrite files outside the repo. Mitigation: centralize all path resolution in one fileSafety guard built on PathGuard.assertInsideRoot, add an fs.realpath + re-check step, and unit-test the escape vectors (§5.2, §20.2). Every endpoint calls the guard; no fs call bypasses it.
  • Destructive overwrite. Save writes straight to disk. Mitigation: writes confined to the guarded root, blocked under .git/, capped by size, and only reachable for files the tree already exposed; the dirty-close confirm prevents accidental UI-side loss. (Accepted: this is the requested "fully editable" behavior.)
  • Monaco bundle size / Vite worker setup. ~2MB and worker config can break the build. Mitigation: self-host workers via the documented Vite pattern, verify frontend:build succeeds, lazy-load the editor route so the chunk isn't in the initial load.
  • Large/binary files freezing the UI. Mitigation: server classifies file kind and enforces size caps; oversized/binary text reads return a typed error the UI renders as "too large / binary."
  • HTML "view as page" executing arbitrary local HTML/JS. Mitigation: render in a sandboxed <iframe> pointed at the raw endpoint (relative assets resolve correctly), not inline injection (§17.4). Acceptable for local single-operator content.

Blast radius: new editor backend domain (additive); new /api/editor route registration in routes/index.ts; new DB table + migration; SessionPanel.tsx and SessionSidebar.tsx gain editor-mode wiring (their existing session behavior must stay intact); Icon.tsx, types/index.ts, vite.config.ts, and both package.jsons gain additions. No existing endpoint or model is modified.

Pushback

  • One git-status-style query per directory expand is fine; a full recursive tree read is not — node_modules-sized trees would stall. The spec mandates lazy per-directory listing; do not "simplify" to a single recursive scan.
  • Persisted tabs add a table for modest payoff. You chose persistence; the cost is one migration + one model + two endpoints. If that feels heavy later, the in-memory fallback is a clean downgrade (drop the state endpoints, keep everything else).
  • Consider a read-only first cut. Writes are the entire Level-3 surface. Shipping browse+view first and adding save behind the verified guard would de-risk — but you explicitly want fully-editable now, so the guard + tests are the gate instead.

Architecture

Backend — new editor domain (mirrors git/)

  • src/routes/editor.ts — thin routes; registered as /api/editor in routes/index.ts.
  • src/controllers/editor/EditorController.ts — thin handlers: validate query/body → call service → ok()/handleEditorError().
  • src/controllers/editor/feature-services/EditorService.ts — listDir, readFile, writeFile, statForRaw; resolves root via ProjectModel, routes every path through the guard.
  • src/controllers/editor/feature-services/EditorStateService.ts — get/upsert persisted tab state via the model.
  • src/controllers/editor/feature-utils/fileSafety.ts — resolveSafePath(root, relPath) (reject abs/.., resolve, realpath re-check), classifyFile(name) → text|image|pdf|html, ignore-list + size + extension constants.
  • src/controllers/editor/feature-utils/EditorError.ts + controllerResponses.ts — typed error + ok/fail/handleEditorError (codes → 400/403/404/413/422/500).
  • src/models/EditorStateModel.ts — all DB access for project_editor_state.
  • src/database/migrations/<ts>_create_project_editor_state.ts — table create.

Endpoints (all under /api/editor, envelope {success,data,error})

  • GET /tree?projectId=&path= → one directory level: [{ name, path, type: "file"|"dir" }], sorted dirs-first, ignore-list applied.
  • GET /file?projectId=&path= → { path, content, kind:"text", tooLarge:false }; rejects binary/oversized with a typed code.
  • GET /raw?projectId=&path= → streams bytes with detected Content-Type, Content-Disposition: inline (images, PDF, HTML-as-page).
  • POST /file {projectId, path, content} → writes text file (guarded, size-capped) → { path, savedAt }.
  • GET /state?projectId= → { openPaths:string[], activePath:string|null }.
  • PUT /state {projectId, openPaths, activePath} → upsert; returns the saved state.

Frontend — new CodeEditor/ feature

  • frontend/src/api/editor.ts — typed fns over the client + a rawFileUrl(projectId,path) helper for <img>/<iframe>/<embed> src.
  • frontend/src/hooks/queries/useEditor.ts — useDirListing, useFileContent, useSaveFile, useEditorState, useSaveEditorState.
  • frontend/src/types/index.ts — add FileTreeEntry, FileContent, EditorState, FileKind.
  • components/CodeEditor/CodeEditorView.tsx — the takeover: file-tree rail (back arrow + project name) + tab bar + active viewer; owns open-tabs state, syncs to useEditorState.
  • components/CodeEditor/FileTree.tsx + FileTreeNode.tsx — lazy expand/collapse, file-type icons.
  • components/CodeEditor/EditorTabs.tsx — tab strip, dirty dot, close (×) with confirm-on-dirty.
  • components/CodeEditor/EditorPane.tsx — Monaco wrapper: language from extension, Emmet init, dirty tracking, Cmd/Ctrl+S.
  • components/CodeEditor/viewers/{ImageViewer,PdfViewer,HtmlViewer}.tsx — image <img>; PDF <embed>/<iframe>; HTML toggle code↔page.
  • components/CodeEditor/editorLanguages.ts — extension → Monaco language id; kind classification mirror of the backend.
  • frontend/src/lib/monaco.ts — loader + MonacoEnvironment worker wiring; vite.config.ts worker config.

Persistence schema — project_editor_state

  • id PK; project_id FK → projects(id) ON DELETE CASCADE, unique; open_paths JSONB (ordered relative paths); active_path TEXT NULL; created_at, updated_at timestamps. Index on project_id (unique covers it).

Change: Executed the plan in an isolated git worktree (../deveasy-code-editor, branch feat/project-code-editor) via parallel sub-agents (backend / frontend-infra / UI), then integrated and verified.

Reason — Execution deviations & discoveries:

  • Pre-existing repo bug found & fixed. .gitignore had an unanchored projects/ rule that also ignored src/controllers/projects/ — so that whole backend domain was never committed (exists only as on-disk files in main). The domain files themselves are left untracked (mirroring main's state) — recommend committing them on main.
  • State validation relaxed to lexical (bug fix). EditorStateService.saveState originally ran the full resolveSafePath (incl. fs.realpath) on each persisted tab path, which hard-failed persistence whenever an open tab was a symlink — and DevEasy injects CLAUDE.md/.claude/* as symlinks. Persisted paths are UI bookmarks, so they are now validated lexically (reject absolute/..) via a new assertRelativePathShape; the authoritative fs guard (incl. symlink realpath) still runs at every read/write.
  • Known tradeoff — symlinked config not openable. The read/write symlink-escape guard (spec mitigation) blocks opening DevEasy's injected CLAUDE.md/.claude/* because they realpath outside the project root. Kept the safe default rather than weaken it. Follow-up: allowlist the managed-config target so those specific files can be opened.
  • Lazy-loaded the editor. CodeEditorView is React.lazy + Suspense in SessionPanel, so Monaco (~3.4 MB) is split into its own on-demand chunk; the initial bundle stayed ~620 kB.
  • Isolated test environment. Worktree points at its own DB deveasy_editor (all 4 migrations applied) and at the real projects folder, on backend port 1240 / Vite 5174 — so it runs alongside the main app without touching its DB/migration state.
  • Backend save body limit raised 2mb → 4mb in src/index.ts so a max-size 2 MB text save survives JSON escaping.

Rev 2 - 2026-06-30

Change: Post-merge security-review fixes (merged to main).

Reason — Security review:

  • HIGH — symlink write-escape closed. resolveSafePath for writes only realpath-checked the parent dir, so saving to an existing path that is itself a symlink-out (e.g. injected CLAUDE.md) followed the link and clobbered a file outside the root. Now realpaths the resolved target itself (rejects symlink-out on read & write), falls back to the parent only for new files, and refuses to write THROUGH any final-component symlink (fs.lstat). Verified live: the exploit returns EDITOR_PATH_OUTSIDE_ROOT and the shared file is left intact.
  • MEDIUM — raw endpoint XSS hardening. /api/editor/raw serves HTML/SVG inline same-origin; added Content-Security-Policy: sandbox + X-Content-Type-Options: nosniff so direct navigation can't run scripts or reach app cookies/APIs. Images (<img>) and PDF still render.

Updated Done criteria: write-through-symlink and existing-symlink-out writes are rejected (2 new vitest, 45 total pass); raw responses carry the sandbox CSP + nosniff.
