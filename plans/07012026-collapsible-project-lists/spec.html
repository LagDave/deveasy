Collapsible Project Lists

Small

Why

The "Sessions by project" sidebar lists every project's sessions at full height. With several projects the rail grows long and the project the operator cares about scrolls out of view. Letting each project collapse — and remembering that choice — keeps the rail compact across reloads.

What

Each project group header becomes a toggle: a chevron plus the project name. Clicking it collapses the project's git line and session list, leaving only the header row. Collapse state is per-project and persisted in localStorage, so it survives a page reload. The folder and "new session" action buttons keep working and do not trigger the toggle.

Context

Relevant Files

  • frontend/src/components/Session/SessionSidebar.tsx - renders the project groups; the only file changed.
  • frontend/src/components/ui/Icon.tsx - already exposes chevronRight and chevronDown icons; no change needed.

Patterns To Follow

  • Collapse state is client-only ephemeral UI state local to the sidebar — a useState in the component, persisted to localStorage. Not Context, not Zustand (Constitution §15.2/§15.3: this state is local to one component, not cross-cutting).
  • Buttons use the existing btn btn-ghost / icon-button classes already in the file.
  • localStorage reads/writes are wrapped in try/catch so a blocked storage API never breaks render (Constitution §16.2: never swallow into a broken UI — here failure is non-fatal and ignored deliberately).

Reference file: frontend/src/components/Session/SessionSidebar.tsx - the change matches the file's own existing structure and Tailwind idiom.

Constraints

Must

  • Persist collapse state per project in localStorage under one namespaced key.
  • Default to expanded for projects with no stored state.
  • Keep folder + new-session buttons clickable without toggling collapse.
  • No console.* (§17.1), no any (§17.2).

Must Not

  • No new dependencies.
  • No new state-management library (§15.4).
  • Do not modify unrelated components or the API/hook layers.

Out Of Scope

  • A "collapse all / expand all" control.
  • Server-side persistence of collapse state.
  • Animated height transition on collapse (instant show/hide is enough).

Risk

Level: 1 (Suggestion / low risk)

Risks Identified

  • Stale ids accumulate in localStorage when projects are deleted. Mitigation: harmless — collapsed ids that no longer match a project are simply never read; the stored array stays tiny.
  • localStorage unavailable (private mode / blocked). Mitigation: reads/writes are in try/catch; feature degrades to in-memory toggle for the session.

Blast radius: SessionSidebar is rendered by the Sessions panel only. No other consumer of the changed file. No API or type change.

Pushback

  • None. Local UI state with localStorage persistence is the simplest correct approach for a per-client preference; promoting it to Context or the server would be over-engineering for a cosmetic toggle.
